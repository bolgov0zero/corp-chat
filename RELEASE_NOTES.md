Фикс: при невалидном токене мобильный PWA-клиент больше не показывает пустой экран чатов

При открытии приложения наличие сессии теперь проверяется через HTTP и WebSocket до того как показывается главный экран. Если сервер не отвечает по HTTP (сетевая ошибка), главный экран откладывается до подтверждения WebSocket-соединения: при получении кода 1008 (токен недействителен) сразу показывается страница авторизации. Добавлено версионирование URL app.js и style.css для принудительного обновления SW-кэша.
