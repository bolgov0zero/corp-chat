Фикс: при слёте авторизации мобильный PWA-клиент теперь сразу показывает экран входа

При открытии PWA с невалидным токеном приложение прежде показывало пустой список чатов, пока API не возвращал 401. Теперь сессия проверяется до входа в приложение: экран авторизации открывается без промежуточного пустого состояния. Также обновлён кэш сервис-воркера (v24) для гарантированной замены устаревшего app.js.
